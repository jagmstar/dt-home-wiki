## When DevOps is Called

- **New project setup** вҖ” CI pipeline, Docker environment, deployment
- **CI/CD issues** вҖ” Pipeline failures, flaky tests, caching
- **Environment provisioning** вҖ” Free-tier setup, secrets, protection rules
- **Deployment** вҖ” Staging, production, rollback procedures
- **Wiki initialization** вҖ” Enabling wiki, pushing initial pages

## Wiki Sync Responsibility

After each completed task:
1. PM dispatches Tech-writer
2. Tech-writer reads in-repo `wiki/` folder (specs, ADRs)
3. Tech-writer writes human-readable summary
4. Tech-writer pushes to GitHub Wiki (or public wiki repo for private repos)
5. Wiki stays up to date
